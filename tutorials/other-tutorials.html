1. Git & GitHub Basics

Step 1 — Version Control

Git is a version control system for tracking changes in computer files and coordinating work on those files among multiple people. GitHub is a platform for hosting Git repositories.

Git Commands 
# Initialize a new Git repository
git init

# Clone an existing repository
git clone https://github.com/username/project.git

# Check status of changes
git status

# Add files to staging area
git add .

# Commit changes
git commit -m "Add new feature"

# Push to remote repository
git push origin main

# Create a new branch
git checkout -b feature-branch

# Switch branches
git checkout main

# Merge branches
git merge feature-branch

# View commit history
git log --oneline

3. Command Line Basics

Step 3 — Terminal Commands

The command line (terminal) gives you powerful control over your computer. Here are essential commands every developer should know:

Terminal 
# Navigation
pwd                 Print working directory
cd folder           Change directory
cd ..               Go up one directory
ls                  List files
ls -la              List all files (including hidden)

# File Operations
touch file.txt      Create empty file
mkdir folder        Create directory
rm file             Remove file
rm -rf folder       Remove directory (force)
cp file1 file2      Copy file
mv file1 file2      Move/rename file
cat file            Display file content
less file           View file page by page

# System
clear               Clear terminal
whoami              Current user
hostname            Computer name
ps aux              Running processes
kill PID            Kill process
top                 Task manager (Mac/Linux)
tasklist            Task manager (Windows)

# Network
ping google.com     Test connection
curl http://url     Make HTTP request
wget http://url     Download file
ssh user@host       SSH connection

4. NPM Package Manager

Step 4 — Node Package Management

NPM (Node Package Manager) is used to install, manage, and share JavaScript packages. It's essential for modern web development.

NPM 
# Initialize new project
npm init -y

# Install packages
npm install package-name
npm i package-name          # Shortcut
npm install -g package      # Global install

# Save as dependency
npm install --save package  # production
npm install --save-dev package  # development

# Uninstall packages
npm uninstall package

# Update packages
npm update package
npm update                  # Update all

# List installed packages
npm list
npm list --depth=0          # Top level only

# Run scripts (from package.json)
npm run dev
npm run build
npm run test

# Clear cache
npm cache clean --force

# Check for outdated packages
npm outdated

# Security audit
npm audit
npm audit fix

5. CSS Flexbox Crash Course

Step 5 — Modern Layouts

Flexbox is a one-dimensional layout method that makes it easy to align items in rows or columns. Perfect for responsive design!

CSS 
# Container Properties
.container {
  display: flex;
  
  # Main axis alignment
  justify-content: flex-start;    # Default
  justify-content: flex-end;
  justify-content: center;
  justify-content: space-between;
  justify-content: space-around;
  
  # Cross axis alignment
  align-items: stretch;           # Default
  align-items: flex-start;
  align-items: flex-end;
  align-items: center;
  
  # Direction
  flex-direction: row;            # Default
  flex-direction: column;
  
  # Wrap
  flex-wrap: nowrap;              # Default
  flex-wrap: wrap;
}

# Item Properties
.item {
  # Size
  flex: 1;                        # grow shrink basis
  flex-grow: 1;                   # How much to grow
  flex-shrink: 1;                 # How much to shrink
  flex-basis: 200px;              # Initial size
  
  # Order
  order: 1;                       # Reorder items
  
  # Self alignment
  align-self: center;
}

6. CSS Grid Layout

Step 6 — Two-Dimensional Layouts

CSS Grid is a two-dimensional layout system that works with rows and columns. Perfect for complex layouts!

CSS 
# Container Setup
.container {
  display: grid;
  
  # Define columns
  grid-template-columns: 200px 200px 200px;
  grid-template-columns: repeat(3, 1fr);
  grid-template-columns: 1fr 2fr 1fr;
  
  # Define rows
  grid-template-rows: 100px 200px 100px;
  
  # Gap between cells
  gap: 20px;
  grid-gap: 20px;               # Old syntax
  
  # Named areas
  grid-template-areas: 
    "header header header"
    "sidebar content content"
    "footer footer footer";
}

# Item Placement
.item {
  # Position by line numbers
  grid-column-start: 1;
  grid-column-end: 3;
  grid-row-start: 1;
  grid-row-end: 2;
  
  # Shorthand
  grid-column: 1 / 3;
  grid-row: 1 / 2;
  
  # Position by area names
  grid-area: header;
}

7. Responsive Design Media Queries

Step 7 — Mobile-First Design

Media queries make your website responsive across different screen sizes. Follow mobile-first approach for best results.

CSS 
# Mobile First Approach

# Base styles (mobile)
body {
  font-size: 16px;
  padding: 16px;
}

# Tablet (min-width: 768px)
@media (min-width: 768px) {
  body {
    font-size: 18px;
    padding: 24px;
  }
  
  .container {
    display: grid;
    grid-template-columns: repeat(2, 1fr);
  }
}

# Desktop (min-width: 1024px)
@media (min-width: 1024px) {
  body {
    font-size: 20px;
  }
  
  .container {
    grid-template-columns: repeat(3, 1fr);
  }
}

# Large Desktop (min-width: 1440px)
@media (min-width: 1440px) {
  .container {
    max-width: 1400px;
    margin: 0 auto;
  }
}

# Common Breakpoints
# Phone: 320px - 480px
# Tablet: 768px - 1024px
# Laptop: 1024px - 1440px
# Desktop: 1440px+
